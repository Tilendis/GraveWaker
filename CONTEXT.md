# CONTEXT — GraveWaker（数字掘墓人）

## 一句话

把散落在各平台的数字收藏"复活"为可检索、可关联的交互式知识地图，让过去的自己和现在的自己对话。

---

## 问题域

### 核心痛点

用户日常在微信、B站、浏览器等平台大量收藏内容（"以后再看"），但收藏夹最终沦为数字垃圾堆——内容从未被再次打开、整理或内化为知识。这不是技术问题，是**人的注意力与惰性问题**。

### 目标用户画像

- 每天大量接触数字内容，知识获取高度依赖互联网
- 没有知识管理习惯，也没有时间精力培养

### 要解决的三个子问题

1. **收集** — 将各平台收藏内容（文本+元数据）收拢到一个本地索引
2. **连接** — 让用户发现收藏之间的主题关联，建立知识地图
3. **唤醒** — 通过主动搜索（钩子）和被动推送（报告），让用户周期性地重新"看见"自己的收藏

---

## 两阶段激活模型

### Phase 1：连接型激活（MVP）

用户带着问题来 → 语义搜索 → 返回相关收藏的位置（平台 + 搜索关键词） → 用户在知识地图上看到关联 → 去原始平台找到内容。

### Phase 2：洞察型激活

系统主动分析 → 发现趋势/变化 → 定期报告推送 → 激发用户的主动搜索 → 回流到 Phase 1。

两阶段共享同一套数据索引，Phase 2 只是 Phase 1 数据上的一层聚合分析。

## 数据源（按优先级）

| 优先级 | 平台 | 内容类型 | 获取方式 | 规模估算 |
|--------|------|---------|---------|---------|
| P0 | B站 | 视频元数据（标题/描述/标签） | 公开 API | ~1000 条 |
| P0 | 浏览器书签 | URL + 标题 | 手动导出 HTML | ~200 条 |
| P1 | 本地电子书 | PDF 文本 | 本地文件读取 | ~200 本 |
| P1 | 微信收藏 | 公众号文章全文 | TBD（反爬风险高） | ~1000 篇 |
| P2 | QQ/微信聊天文件 | PDF | 本地文件 | ~500 个 |

## 功能需求

### F1: 离线内容索引
- 从各数据源提取文本 + 元数据（标题、日期、来源平台、URL）
- 对文本分段、向量化、存入本地索引
- 支持增量更新（新增收藏时不必重建全量索引）

### F2: 语义搜索
- 用户用自然语言描述需求
- 返回匹配的收藏列表，每条包含：平台、标题/名称、收藏日期、匹配摘要
- 提供"去哪个平台搜什么关键词"的导航建议

### F3: 交互式知识地图
- 可视化展示收藏内容之间的主题关联（节点 = 内容，连线 = 相似主题）
- 地图旁有搜索框，搜索后放大相关局部区域
- 点击节点可查看详情和定位信息

### F4: 定期报告推送
- 周期性（如每周/每月）生成摘要报告
- 内容：新增收藏概览、主题分布变化、跨时间兴趣对比
- 通过邮箱送达用户

## 约束

| 约束 | 详情 |
|------|------|
| **磁盘** | C 盘不允许占用；D 盘 50GB 可用，项目控制在 10GB 以内 |
| **时间** | 每天 ≤1h，周末 ≤2h |
| **人力** | 1 人，非专业开发者，AI 辅助 |
| **技术** | 仅会基础 Python（跑脚本、读 README），不会前端/数据库/运维 |
| **成本** | 零预算，优先免费云 API，后期可转本地模型 |
| **合规** | 不违反平台 ToS（优先使用公开 API 和手动导出，避免高风险爬取） |
| **传播** | 小范围使用即可，无盈利需求 |

## 关键术语表

| 术语 | 定义 |
|------|------|
| **数字收藏** | 用户在各平台保存/收藏/标记的内容，含文章、视频、书签、电子书等 |
| **催化** | 数据 → 信息 → 知识 的转化过程代码化 |
| **连接型激活** | 用户主动搜索 → 发现关联 → 找到内容 |
| **洞察型激活** | 系统主动分析 → 推送报告 → 激发搜索 |
| **钩子** | 让用户重新关注旧收藏的触发机制（搜索框是主动钩子，报告是被动钩子） |
| **知识地图** | 以图结构可视化收藏内容之间主题关联的交互界面 |
| **元数据** | 收藏的标题、来源平台、保存日期、URL 等结构信息 |
| **语义搜索** | 基于向量相似度而非关键词匹配的搜索方式 |
| **内容索引** | 所有收藏文本经过向量化后存储的本地数据库 |

## 非目标（明确不做）

- 不替代任何平台的原生收藏功能
- 不提供内容源文件的云端备份
- 不支持多用户、协作、分享
- 不做移动端 App
- 不爬取需要登录+反爬对抗的平台（Phase 1 不做）
